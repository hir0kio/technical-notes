# Post-compromise security

暗号鍵を自動的に交換する仕組みにより、暗号鍵が危殆化してもその鍵がいずれ正常な鍵と置き換えられることで、プロトコルが自己修復し、危殆化後の通信の機密性を保つ暗号プロトコルの機能。future secrecy。

[Double Ratchet アルゴリズム](double-ratchet-algorithm)では、Alice と Bob がそれぞれ短期鍵（[ラチェット鍵](double-ratchet-algorithm#ラチェット鍵)）を保持して、相手のメッセージに返信するときに自分の短期鍵を更新し、自分の新しい短期鍵と相手の既存の短期鍵で[楕円曲線ディフィー・ヘルマン鍵共有](https://ja.wikipedia.org/wiki/楕円曲線ディフィー・ヘルマン鍵共有)を行って得られた秘密の値をもとに暗号鍵を更新する。これにより、ある時点で暗号鍵が危殆化しても、その暗号鍵が更新されたあとの通信の機密性が保たれる。
