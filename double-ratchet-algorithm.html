Double Ratchet アルゴリズム #

共有した秘密の値をもとに安全にメッセージを暗号化して送受信するためのアルゴリズム。暗号鍵を自動的に交換するラチェッティング1という仕組みにより、暗号鍵が危殆化した場合に過去の通信の機密性を保ち、また危殆化した鍵がいずれ正常な鍵と交換されることで、前方秘匿性と post-compromise security を実現する。

  1. ratchet は「爪車」（逆転止めの爪と組み合わせて、一方向だけに回転するように作られている歯車）の意。

主な概念 #

ラチェット #

ratchet

鍵を入力すると、対応するキー マテリアルを出力して鍵を更新する鍵導出関数。

  • 鍵k+1, キー マテリアルk ← KDF(鍵k)

非対称ラチェット #

asymmetric ratchet

自分のラチェット鍵を更新し、ルート鍵、自分の新しいラチェット鍵、相手のラチェット鍵を入力すると、対応するチェーン鍵を出力してルート鍵を更新するラチェット。

  • 自分のラチェット鍵 ← 鍵生成()
  • DH 出力 ← ECDH(自分のラチェット鍵, 相手のラチェット鍵)
  • ルート鍵k+1, チェーン鍵k ← KDF(ルート鍵k, DH 出力)

対称ラチェット #

symmetric ratchet

チェーン鍵を入力すると、対応するメッセージ鍵を出力してチェーン鍵を更新するラチェット。

  • チェーン鍵k+1, メッセージ鍵k ← KDF(チェーン鍵k)

ラチェット鍵 #

ratchet key

楕円曲線暗号鍵。非対称ラチェットを更新するときに新しく作成し、交換する。

ルート鍵 #

root key

秘密の値。非対称ラチェットに入力し、対応するチェーン鍵を導出する。初期ハンドシェイクで共有した値を最初のルート鍵とする。

チェーン鍵 #

chain key

秘密の値。対称ラチェットに入力し、対応するメッセージ鍵を導出する。

暗号化用のメッセージ鍵の導出に使うチェーン鍵は送信チェーン鍵、復号用のメッセージ鍵の導出に使うチェーン鍵は受信チェーン鍵と呼ばれる。

メッセージ鍵 #

message key

メッセージを暗号化・復号するための対称鍵。対称ラチェットから導出する。

それぞれのメッセージに固有であり、別のメッセージを同じメッセージ鍵で暗号化することはできない。

鍵のライフサイクル #

鍵ライフサイクル
ラチェット鍵鍵の所有者が非対称ラチェットを更新するまで。
ルート鍵非対称ラチェットを更新するまで。
チェーン鍵非対称ラチェットまたは対称ラチェットを更新するまで。
メッセージ鍵メッセージの暗号化・復号に使ったら削除する。

役割 #

  • Alice は、Bob にメッセージを送る。ただし、メッセージを送るとき Bob はオフラインである場合がある。
  • Bob は、Alice からのメッセージを受け取って Alice に返信する。ただし、返信を送るとき Alice はオフラインである場合がある。
  • サーバ は、Alice と Bob が相手に送ったデータを一時的に保持し、相手がオンラインに復帰するとこのデータを配送する。

手順 #

Signal プロトコルは、主に次の 3 段階から構成される。

  1. 初期ハンドシェイク。X3DH 鍵共有プロトコルを使ってルート鍵を共有する。
  2. 非対称ラチェット ステージ。ルート鍵と Alice と Bob のラチェット鍵から対応するチェーン鍵を生成する。
  3. 対称ラチェット ステージ。チェーン鍵から対応するメッセージ鍵を生成する。

Double Ratchet アルゴリズムは「非対称ラチェット ステージ」と「対称ラチェット ステージ」を担当する。

1. 初期ハンドシェイク #

  1. Alice は X3DH 鍵合意プロトコルを使って Bob と秘密の値を共有する。この値をルート鍵0とする。

2. Alice: Bob にメッセージを送信 #

Alice は Bob にメッセージを送信する。

2.1. 非対称ラチェット ステージ #

  1. Alice は新しいラチェット鍵を作成する。
    • Alice のラチェット鍵 ← 鍵生成()
  2. ルート鍵0、自分のラチェット鍵、Bob の前鍵を非対称ラチェットに入力し、対応する送信チェーン鍵0, 0と次のルート鍵1を得る。
    • DH 出力 ← ECDH(Alice のラチェット鍵, Bob の前鍵)
    • ルート鍵1, 送信チェーン鍵0, 0 ← KDF(ルート鍵0, DH 出力)

2.2. 対称ラチェット ステージ #

  1. 送信チェーン鍵0, 0を対称ラチェットに入力し、対応するメッセージ鍵0, 0と次の送信チェーン鍵0, 1を得る。
    • 送信チェーン鍵0, 1, メッセージ鍵0, 0 ← KDF(送信チェーン鍵0, 0)
  2. メッセージ鍵0, 0で最初のメッセージを暗号化する。
  3. さらにメッセージを送る場合は、
    1. 送信チェーン鍵0, 1を対称ラチェットに入力し、対応するメッセージ鍵0, 1と次の送信チェーン鍵0, 2を得る。
      • 送信チェーン鍵0, 2, メッセージ鍵0, 1 ← KDF(送信チェーン鍵0, 1)
    2. メッセージ鍵0, 1で次のメッセージを暗号化する。
    3. さらにメッセージを送る場合は、
      1. 送信チェーン鍵0, 2を対称ラチェットに入力し、対応するメッセージ鍵0, 2と次の送信チェーン鍵0, 3を得る。
        • 送信チェーン鍵0, 3, メッセージ鍵0, 2 ← KDF(送信チェーン鍵0, 2)
      2. メッセージ鍵0, 2で次のメッセージを暗号化する。
      3. ......
  4. すべてのメッセージを暗号化するまで繰り返す。
  1. 暗号化したメッセージと自分のラチェット鍵をサーバに送信する。

ルート鍵またはチェーン鍵を更新したあとは、前方秘匿性のため古い鍵を削除する。

3. Bob: Alice からのメッセージを受信 #

Bob は Alice からのメッセージを受信する。

3.1. 非対称ラチェット ステージ #

  1. Bob は Alice の暗号化されたメッセージとラチェット鍵をサーバから受け取る。
  2. ルート鍵0、自分の前鍵、Alice のラチェット鍵を非対称ラチェットに入力し、対応する受信チェーン鍵0, 0と次のルート鍵1を得る。
    • DH 出力 ← ECDH(Bob の前鍵, Alice のラチェット鍵)
    • ルート鍵1, 受信チェーン鍵0, 0 ← KDF(ルート鍵0, DH 出力)

3.2. 対称ラチェット ステージ #

  1. 受信チェーン鍵0, 0を対称ラチェットに入力し、対応するメッセージ鍵0, 0と次の受信チェーン鍵0, 1を得る。
    • 受信チェーン鍵0, 1, メッセージ鍵0, 0 ← KDF(受信チェーン鍵0, 0)
  2. メッセージ鍵0, 0で Alice の最初のメッセージを復号する。
  3. まだ受け取ったメッセージがある場合は、
    1. 受信チェーン鍵0, 1を対称ラチェットに入力し、対応するメッセージ鍵0, 1と次の受信チェーン鍵0, 2を得る。
      • 受信チェーン鍵0, 2, メッセージ鍵0, 1 ← KDF(受信チェーン鍵0, 1)
    2. メッセージ鍵0, 1で Alice の次のメッセージを復号する。
    3. まだ受け取ったメッセージがある場合は、
      1. 受信チェーン鍵0, 2を対称ラチェットに入力し、対応するメッセージ鍵0, 2と次の受信チェーン鍵0, 3を得る。
        • 受信チェーン鍵0, 3, メッセージ鍵0, 2 ← KDF(受信チェーン鍵0, 2)
      2. メッセージ鍵0, 2で Alice の次のメッセージを復号する。
      3. ......
  4. すべてのメッセージを復号するまで繰り返す。

4. Bob: Alice にメッセージを送信 #

Alice から受け取ったメッセージに Bob が返信する。

4.1. 非対称ラチェット ステージ #

  1. Bob は新しいラチェット鍵を作成する。
    • Bob のラチェット鍵 ← 鍵生成()
  2. ルート鍵1、自分のラチェット鍵、Alice のラチェット鍵を非対称ラチェットに入力し、対応する送信チェーン鍵1, 0と次のルート鍵2を得る。
    • DH 出力' ← ECDH(Bob のラチェット鍵, Alice のラチェット鍵)
    • ルート鍵2, 送信チェーン鍵1, 0 ← KDF(ルート鍵1, DH 出力')

4.2. 対称ラチェット ステージ #

  1. 送信チェーン鍵1, 0を対称ラチェットに入力し、対応するメッセージ鍵1, 0と次の送信チェーン鍵1, 1を得る。
    • 送信チェーン鍵1, 1, メッセージ鍵1, 0 ← KDF(送信チェーン鍵1, 0)
  2. メッセージ鍵1, 0で最初のメッセージを暗号化する。
  3. さらにメッセージを送る場合は、
    1. 送信チェーン鍵1, 1を対称ラチェットに入力し、対応するメッセージ鍵1, 1と次の送信チェーン鍵1, 2を得る。
      • 送信チェーン鍵1, 2, メッセージ鍵1, 1 ← KDF(送信チェーン鍵1, 1)
    2. メッセージ鍵1, 1で次のメッセージを暗号化する。
    3. さらにメッセージを送る場合は、
      1. 送信チェーン鍵1, 2を対称ラチェットに入力し、対応するメッセージ鍵1, 2と次の送信チェーン鍵1, 3を得る。
        • 送信チェーン鍵1, 3, メッセージ鍵1, 2 ← KDF(送信チェーン鍵1, 2)
      2. メッセージ鍵1, 2で次のメッセージを暗号化する。
      3. ......
  4. すべてのメッセージを暗号化するまで繰り返す。
  1. 暗号化したメッセージと自分のラチェット鍵をサーバに送信する。

5. Alice: Bob からのメッセージを受信 #

Alice は Bob からの返信を受信する。

5.1. 非対称ラチェット ステージ #

  1. Alice は Bob の暗号化されたメッセージとラチェット鍵をサーバから受け取る。
  2. ルート鍵1、自分のラチェット鍵、Bob のラチェット鍵を非対称ラチェットに入力し、対応する受信チェーン鍵1, 0と次のルート鍵2を得る。
    • DH 出力' ← ECDH(Alice のラチェット鍵, Bob のラチェット鍵)
    • ルート鍵2, 受信チェーン鍵1, 0 ← KDF(ルート鍵1, DH 出力')

5.2. 対称ラチェット ステージ #

  1. 受信チェーン鍵1, 0を対称ラチェットに入力し、対応するメッセージ鍵1, 0と次の受信チェーン鍵1, 1を得る。
    • 受信チェーン鍵1, 1, メッセージ鍵1, 0 ← KDF(受信チェーン鍵1, 0)
  2. メッセージ鍵1, 0で Bob の最初のメッセージを復号する。
  3. まだ受け取ったメッセージがある場合は、
    1. 受信チェーン鍵1, 1を対称ラチェットに入力し、対応するメッセージ鍵1, 1と次の受信チェーン鍵1, 2を得る。
      • 受信チェーン鍵1, 2, メッセージ鍵1, 1 ← KDF(受信チェーン鍵1, 1)
    2. メッセージ鍵1, 1で Bob の次のメッセージを復号する。
    3. まだ受け取ったメッセージがある場合は、
      1. 受信チェーン鍵1, 2を対称ラチェットに入力し、対応するメッセージ鍵1, 2と次の受信チェーン鍵1, 3を得る。
        • 受信チェーン鍵1, 3, メッセージ鍵1, 2 ← KDF(受信チェーン鍵1, 2)
      2. メッセージ鍵1, 2で Bob の次のメッセージを復号する。
      3. ......
  4. すべてのメッセージを復号するまで繰り返す。

考察 #

アルゴリズムの設計から、もしルート鍵kが危殆化しても、鍵導出関数を reverse することができないためそれ以前のルート鍵k-1, k-2, k-3, ...... の機密性が保たれる上に、Alice と Bob のラチェット鍵の楕円曲線ディフィー・ヘルマン鍵共有が非対称ラチェットにエントロピーを与えるため、ラチェット鍵が削除されている限りその後のルート鍵k+1, k+2, k-3, ...... の機密性も保たれることがわかる。

また、それぞれのメッセージは毎回異なるチェーン鍵を対称ラチェットに入力して導出した固有のメッセージ鍵で暗号化されているため、（a）もしあるチェーン鍵lが危殆化してもそれ以前のメッセージ鍵l-1, l-2, l-3 ...... の機密性が保たれること、（b）もしあるメッセージ鍵が危殆化してもほかのメッセージ鍵の機密性が保たれることがわかる。

実装 #

メッセージの送信 #

let ルート鍵;

function メッセージの送信(
  // 初めてメッセージを送るときは相手の前鍵。
  相手のラチェット鍵,

  平文メッセージ
) {
  let 暗号化済みメッセージ = [];

  // 自分のラチェット鍵を更新する。
  let 自分のラチェット鍵 = 鍵生成();

  // 非対称ラチェットを更新する。
  let DH_出力 = ECDH(自分のラチェット鍵, 相手のラチェット鍵);
  let [新しいルート鍵, 送信チェーン鍵] = KDF(ルート鍵, DH_出力);
  ルート鍵 = 新しいルート鍵;

  for (let m of 平文メッセージ) {
    // 対称ラチェットを更新する。
    let [新しい送信チェーン鍵, メッセージ鍵] = KDF(送信チェーン鍵);
    送信チェーン鍵 = 新しい送信チェーン鍵;

    // メッセージ m を暗号化する。
    暗号化済みメッセージ.push(暗号化(m, メッセージ鍵));
  }

  return {
    自分のラチェット鍵,
    暗号化済みメッセージ,
  };
}

メッセージの受信 #

let ルート鍵;

// 初めてメッセージを受け取るときは自分の前鍵。
let 自分のラチェット鍵;

function メッセージの受信(相手のラチェット鍵, 暗号化済みメッセージ) {
  let 平文メッセージ = [];

  // 非対称ラチェットを更新する。
  let DH_出力 = ECDH(自分のラチェット鍵, 相手のラチェット鍵);
  let [新しいルート鍵, 受信チェーン鍵] = KDF(ルート鍵, DH_出力);
  ルート鍵 = 新しいルート鍵;

  for (let m of 暗号化済みメッセージ) {
    // 対称ラチェットを更新する。
    let [新しい受信チェーン鍵, メッセージ鍵] = KDF(受信チェーン鍵);
    受信チェーン鍵 = 新しい受信チェーン鍵;

    // 暗号化済みメッセージ m を復号する。
    平文メッセージ.push(復号(m, メッセージ鍵));
  }

  return {
    自分のメッセージ鍵,
    平文メッセージ,
  };
}
